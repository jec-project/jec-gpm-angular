settings Project Structure

The current section describes the architecture of an Angular project deployed in an Entreprise JavaScript Project (EJP).

GlassCat EJP Structure

EJPs have been designed to clearly separate the presentation (front-side) from the business logic (back-side) of your application. The presentation is located in the webapp folder, while the src directory contains the business logic: 

project
   │
   ├─src
   ├─typings
   ├─webapp
   └─tsconfig.json

The tsconfig.json file ad the typings folder define TypeScript configuration settings.

The Webapp Folder

webapp
   │
   ├─app
   ├─node_modules
   ├─styles
   ├─views
   ├─WEB-INF
   ├─index.html
   └─package.json

Angular Application Folder

app
 │
 ├─business
 ├─components
 ├─modules
 ├─services
 ├─templates
 ├─main.ts
 └─systemjs.config.js

Source Folder

The scr folder usually contains back-side components, such as jslets and services. JEC specification recommends to separate front-side and back-side components in different EJPs; so, you should never add components in the source scr of a JEC Angular app. 

src
 │
 └─jslets
     ├─AngularApp.ts
     └─Status.ts

The AngularApp jslet provides a convenient way to control the visibility of an application by setting the available routes: 

@WebJslet({{'{'}}
  name: "AngularApp",
  urlPatterns: [
    "/app",
    "/app/welcome",
    "/app/project-structure",
    "/app/lazy-loading/*"
  ],
  template: "/views/angular-app.ejs"
{{'}'}})
